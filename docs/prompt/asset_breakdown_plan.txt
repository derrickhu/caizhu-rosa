彩珠五连美术资源拆分计划

阶段 1：确认原型图
先用 6 张 9:16 原型图确认整体方向：首页、经典对局、关卡对局、关卡选择、排行榜、皮肤/背景页。确认后不直接把整张原型当运行资源，而是拆成可复用组件。

阶段 2：通用 UI 组件
资源清单：
1. 主按钮：绿色、蓝色、橙色、紫色四种状态，正常/按下/禁用。
2. 圆形入口按钮：排行榜、皮肤、设置、福利、返回、关闭。
3. 面板底板：HUD、弹窗、排行榜列表、皮肤卡片、关卡节点卡片。
4. 进度条：蓝色未满、绿色达成、黄色奖励。
5. 星星、锁、奖牌、皇冠等小图标。
生成要求：
尽量无文字，透明 PNG，文字由代码绘制。

阶段 3：棋子皮肤套装
每套 7 色球，颜色索引保持当前规则：
0 红、1 黄、2 蓝、3 绿、4 紫、5 橙/粉、6 青。
建议套装：
1. 经典高光珠：默认。
2. 果冻珠：半透明、Q 弹。
3. 宝石珠：切面高光。
4. 糖果珠：糖霜/甜品感。
5. 海洋珠：贝壳、水滴、泡泡元素。
每套输出 7 张透明 PNG，后续路径建议：
minigame/images/skins/{skinId}/orb_0.png ... orb_6.png

阶段 4：背景图
建议背景：
1. bg_home_candy_ocean.jpg：首页鲜艳主背景。
2. bg_classic_blue.jpg：经典模式清晰深蓝背景。
3. bg_level_coral.jpg：关卡模式明亮珊瑚/海底背景。
4. bg_board_ocean.jpg：可解锁对局背景。
5. bg_board_candy.jpg：可解锁对局背景。
6. bg_board_starry.jpg：可解锁对局背景。
运行路径建议：minigame/images/backgrounds/{bgId}.jpg。

阶段 5：道具图标
当前代码支持横向雪碧图 prop_icons.png。后续可继续用 80x80 单行 5 帧：
落点预览、撤回一步、移除珠子、重选颜色、续命。
若想扩展到更多道具，需同步修改 iconLoader 的索引与 prop_icons.json。

阶段 6：集成原则
1. 生成原型图放到外部 game_assets 原始目录，不直接进包。
2. 正式资源放到 minigame/images/，构建不会自动复制，需要路径稳定。
3. 大图用 JPG/WebP 前需确认微信兼容，优先 JPG/PNG。
4. 所有正式 UI 资源尽量不带文字。
